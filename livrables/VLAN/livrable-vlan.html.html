1. Introduction

1.1 Contexte du projet

Le cabinet Comptafinance démarre son infrastructure informatique et souhaite segmenter ses deux services principaux, Accueil et Gestion, en utilisant la technologie VLAN pour améliorer la sécurité et l’organisation du réseau. [file:1]

1.2 Objectifs

  • Mettre en place deux VLANs distincts pour séparer les services Accueil et Gestion. [file:1]
  • Configurer un serveur DHCP centralisé pour l’attribution automatique des adresses IP. [file:1]
  • Établir la communication inter-VLAN via un routeur (router-on-a-stick). [file:1]
  • Partager une imprimante réseau entre les deux VLANs. [file:1]
  • Simuler un accès Internet via le routeur. [file:1]

1.3 Équipements utilisés

Équipement Modèle / Type Quantité
Routeur Cisco (avec sous-interfaces 802.1Q) 1
Switch Cisco 2960 (compatible 802.1Q) 1
Serveur DHCP Windows / Linux Server 1
PC Accueil Postes clients VLAN 10 2
PC Gestion Postes clients VLAN 20 3
Imprimante réseau Partagée entre VLANs 1

2. Architecture réseau

2.1 Description générale

L’architecture est construite autour d’un switch central qui segmente le réseau en deux VLANs et d’un routeur qui assure le routage inter-VLAN et la sortie vers Internet. [file:1]

  • Routeur : point d’accès vers Internet et routage inter-VLAN via sous-interfaces 802.1Q. [file:1]
  • Switch central : séparation logique des VLANs et distribution des ports utilisateurs. [file:1]
  • VLAN 10 (Accueil) : 2 PC + serveur DHCP + imprimante réseau. [file:1]
  • VLAN 20 (Gestion) : 3 PC utilisateurs. [file:1]
  • Imprimante : située dans le VLAN 10, accessible depuis les deux VLANs via le routeur. [file:1]

4. Configuration du switch

4.1 Création des VLANs

Création VLAN 10 (Accueil)
Switch> enable Switch# configure terminal Switch(config)# vlan 10 Switch(config-vlan)# name ACCUEIL Switch(config-vlan)# exit
Création VLAN 20 (Gestion)
Switch(config)# vlan 20 Switch(config-vlan)# name GESTION Switch(config-vlan)# exit

4.2 Configuration du port trunk vers le routeur

Le port trunk transporte les VLAN 10 et 20 entre le switch et le routeur via 802.1Q. [file:1]

Port trunk (exemple Gi0/1)
Switch(config)# interface GigabitEthernet 0/1 Switch(config-if)# switchport mode trunk Switch(config-if)# switchport trunk allowed vlan 10,20 Switch(config-if)# description LIAISON_TRUNK_VERS_ROUTEUR Switch(config-if)# exit

4.3 Assignation des ports en mode access

Les numéros de ports sont à adapter selon le câblage réel. [file:1]

Ports VLAN 10 (Accueil)
Switch(config)# interface range FastEthernet 0/2-5 Switch(config-if-range)# switchport mode access Switch(config-if-range)# switchport access vlan 10 Switch(config-if-range)# description VLAN_ACCUEIL Switch(config-if-range)# exit
Ports VLAN 20 (Gestion)
Switch(config)# interface range FastEthernet 0/6-10 Switch(config-if-range)# switchport mode access Switch(config-if-range)# switchport access vlan 20 Switch(config-if-range)# description VLAN_GESTION Switch(config-if-range)# exit

4.4 Sauvegarde & vérifications

Sauvegarde configuration
Switch(config)# end Switch# write memory ! ou Switch# copy running-config startup-config
Commandes de vérification
show vlan brief show interfaces trunk show interfaces fa0/2 switchport

5. Configuration du routeur

5.1 Activation de l’interface physique

Interface principale
Router> enable Router# configure terminal Router(config)# interface GigabitEthernet 0/0 Router(config-if)# no shutdown Router(config-if)# description INTERFACE_TRUNK Router(config-if)# exit

5.2 Sous-interface VLAN 10 (Accueil)

Gi0/0.10
Router(config)# interface GigabitEthernet 0/0.10 Router(config-subif)# encapsulation dot1Q 10 Router(config-subif)# ip address 192.168.10.1 255.255.255.0 Router(config-subif)# description GATEWAY_VLAN_ACCUEIL Router(config-subif)# exit

5.3 Sous-interface VLAN 20 (Gestion) + relais DHCP

Gi0/0.20 + ip helper-address
Router(config)# interface GigabitEthernet 0/0.20 Router(config-subif)# encapsulation dot1Q 20 Router(config-subif)# ip address 192.168.20.1 255.255.255.0 Router(config-subif)# description GATEWAY_VLAN_GESTION Router(config-subif)# ip helper-address 192.168.10.10 Router(config-subif)# exit

La commande ip helper-address redirige les requêtes DHCP broadcast du VLAN 20 vers le serveur DHCP du VLAN 10 (192.168.10.10). [file:1]

5.4 Sauvegarde & vérifications

Sauvegarde configuration
Router(config)# end Router# write memory
Commandes de vérification
show ip interface brief show ip route ping 192.168.10.1 ping 192.168.20.1

7. Tests et validation

7.1 Connectivité intra-VLAN

  • Depuis PC1 (VLAN 10) : ping 192.168.10.10 (serveur DHCP), ping 192.168.10.15 (imprimante). [file:1]
  • Depuis PC4 (VLAN 20) : ping 192.168.20.21 (PC5). [file:1]

7.2 Routage inter-VLAN

  • Depuis PC1 (VLAN 10) : ping 192.168.20.1 (gateway VLAN 20). [file:1]
  • Depuis PC4 (VLAN 20) : ping 192.168.10.1 (gateway VLAN 10), ping 192.168.10.15 (imprimante). [file:1]

7.3 Vérification DHCP

  • Configurer les PC en DHCP puis ipconfig /renew (Windows) ou dhclient (Linux). [file:1]
  • Vérifier IP, masque, passerelle et serveur DHCP (192.168.10.10). [file:1]

7.4 Accès à l’imprimante

  • Depuis PC4 (VLAN 20) : ping 192.168.10.15 puis ouverture de http://192.168.10.15 si l’imprimante a une interface web. [file:1]
  • Ajout de l’imprimante en réseau via son adresse IP. [file:1]

7.5 Checklist finale

  • VLANs 10 et 20 créés sur le switch. [file:1]
  • Trunk fonctionnel entre switch et routeur. [file:1]
  • Sous-interfaces du routeur actives. [file:1]
  • Serveur DHCP distribue des IP aux deux VLANs. [file:1]
  • Communication intra-VLAN opérationnelle. [file:1]
  • Communication inter-VLAN opérationnelle. [file:1]
  • Imprimante accessible depuis les deux VLANs. [file:1]
  • Configuration sauvegardée sur switch et routeur. [file:1]

8. Dépannage

8.1 Les PC ne reçoivent pas d’adresse IP

  • Vérifier que le serveur DHCP est démarré. [file:1]
  • Contrôler ip helper-address sur Gi0/0.20. [file:1]
  • Vérifier les pools DHCP. [file:1]
  • S’assurer que les ports sont dans le bon VLAN. [file:1]

8.2 Pas de communication inter-VLAN

  • Vérifier que Gi0/0 est up (no shutdown). [file:1]
  • Contrôler la configuration des sous-interfaces. [file:1]
  • Vérifier l’encapsulation dot1Q. [file:1]
  • Tester les passerelles en ping. [file:1]

8.3 Le trunk ne fonctionne pas

  • Vérifier le câblage entre switch et routeur. [file:1]
  • S’assurer que le port est bien en mode trunk. [file:1]
  • Vérifier que les VLANs 10 et 20 sont autorisés. [file:1]

9. Conclusion

Ce projet met en œuvre une infrastructure réseau segmentée en VLAN pour le cabinet Comptafinance, avec un routage inter-VLAN fonctionnel et un serveur DHCP centralisé. [file:1]

Les objectifs de séparation des services, d’automatisation de l’adressage IP et de partage de ressources (imprimante) ont été atteints, tout en fournissant une documentation claire pour la maintenance future. [file:1]

Annexe C : Glossaire

  • VLAN (Virtual LAN) : réseau local virtuel permettant de segmenter un réseau physique. [file:1]
  • Trunk : liaison transportant plusieurs VLANs simultanément. [file:1]
  • 802.1Q : protocole standard de marquage des trames VLAN. [file:1]
  • Router-on-a-stick : routage inter-VLAN via sous-interfaces sur un routeur. [file:1]
  • DHCP Relay (ip helper-address) : redirection des requêtes DHCP entre VLANs. [file:1]
  • Access Port : port switch assigné à un seul VLAN. [file:1]
  • Gateway : passerelle par défaut pour sortir du réseau local. [file:1]
